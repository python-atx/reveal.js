Basic Python

  • Python is a high level interpreted programming language

  • high level: We don't interact with memory and data directly, but through well designed interfaces and libraries
  • interpreted: We need an interpreter for each platform that we want our python code to run
  • programming language: Language to communicate and tell the computers what to do

Data Types


Boolean

  • Indicate a True or False state
  • Has a numeric value: True: 1; False: 0

>>> 10 < 11
True
>>> 10 > 11
False
>>> type(10 > 11)
<class 'bool'>
>>> (11 > 10) + 1
2
>>> (10 > 11) - 1
-1

Numbers

  • Represent numeric values

>>> x = 5
>>> y = 11/x
>>> y
2.2
>>> type(y)
< class 'float'>
>>> type(x)
< class 'int'>

String

  • A series of characters
  • Strings are enclosed by either single quote or double quotes

>>> "This is a string"
'This is a string'
>>> 'This is also a string'
'This is also a string'
>>> 'This's a string too...right?'
    File "<stdin>", line 1
       'This's a string too...right?'
             ^
SyntaxError: invalid syntax

  • If a string contains single quote, use double quotes to enclose it, and vice versa
  • Sometimes a string must be escaped, using \ character

  • Many characters can be escaped.
    • \': A single quote
    • \": A double quote
    • \n: A new line
    • \t: A tab

    
>>> "This's a string"
"This's a string"
>>> 'This\'s also a string'
"This's also a string"
>>> "Jon's friend said: \"Hi!\""
'Jon\'s friend said: "Hi!"'
>>> print("Here's a \nNewLine")
Here's a
NewLine

    Mixing data types

    • Do not mix data types types
    • Python usually behaves unexpectedly, or throws an error
    • Convert all variables to one type before using

    Mixing int and float

    • Usually convert both numbers to float
    • Force conversion by using int() or float()
    
>>> 1 + 2.0
3.0
>>> 1 + int(1.2)
2
>>> float(1) + 1.2
2.2

    Mixing numbers and string

    • Usually cause an error
    • You can convert numbers to string using str()
    
>>> '10' + '11'
'1011'
>>> 10 + 11
21
>>> 10 + '11'
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
TypeError: unsupported operand type(s) for +: 'int' and 'str'
>>> 10 + int('11')
21
>>> str(10) + '11'
'1011'

    • Sometimes, mixing is OK
    • Must be careful and anticipate unexpected behavior
    
>>> 12 * 3
36
>>> '12' * 3
'121212'
