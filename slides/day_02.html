Control flows

  • Refering to changing the flow of the program/script
  • So far: Python executes line by line, from top to bottom
  • We can not do any logic:
    • Optionally execute these statements (If/Then branch)
    • Execute these statements instead of those (If/Else branch)
    • Repeat these statements (Loop)

Code block

  • A bunch of statements performing one logic operation is called a code block
  • Statements of the same blocks are grouped together using indentation
  • A code block can contains other code blocks
    • 
code_block_0_statement1
code_block_0_statement2
    codeblock_1_statement1
    codeblock_1_statement2
    codeblock_1_statement3
code_block_0_statement3
code_block_0_statement4

Indentation

  • You can use tab or multiple spaces for indentation
  • Convention: Use 4 spaces
  • You should be consistent: If you use one, stick with one for the entire script
  • Using an IDE like PyCharm helps convert indentation to one style (which is configurable)
  • You can safely use the tab key for all indentation, PyCharm will convert to 4 spaces

If/Then branch

Syntax: 

statement0
statement1
if condition:
    statement2
    statement3
statement4

  • An if clause ends with a collon (:)
  • It is followed by a code block (with indentation)

  • Solve this problem: Optionally execute a code block
  • Does not change the script's normal behavior
  • But do some additional things

print("Don't forget your phone")
print("Don't forget your keys")
if temperature < 70:
    print("Don't forget your jacket either!")
    print("Don't forget your lunch")  # Is this what we want?
print("Don't forget to check mail")
  • Make sure your indentation is correct
  • Do we bring lunch every day? Or only on code days?

If/Else branch

Syntax: 

statement0
if condition:
    statement1
    statement2
else:
    statement3
    statement4
statement5

  • if statement ends with a collon (:)
  • else statement also ends with a collon
  • Each of them is followed by a code block

  • Solve this problem: Execute eitherthis block or that block
  • Have to execute one and exactly one of the two blocks
  • This does change the script behavior
  • Script has to take one of the code path depending on the condition

if temperature < 70:
    print("It's cold out there")
else:
    print("It's hot out there")
print("Dress properly!"")

If/Then vs If/Else

  • We can convert If/Then branch to If/Else branch by using empty block
  • To signal an empty block, use the keyword pass in the empty block
  • The following two examples do exactly the same thing:

if temperature < 70:
    print("It's cold")
                    

if temperature < 70:
    print("It's cold")
else:
    pass

Q: What happens if you have pass within a code block that is not empty?

A: Nothing! pass is ignored

Loop

Syntax: 

while condition:
    statement1
    statement2
    statement3
    update condition
  • Repeat the code block when the condition is True
  • Condition must be updated within the loop
  • A while statement must end with : (collon)
  • Another commonly seen phrase: execute these until condition
  • Meaning: Stop execution when condition is True
  • Also mean: Execute these while condition is not True
  • Translate to Python: 
    while not condition:
    statement_0
    statement_1
    update condition
  • condition must be updated within the loop
  • Another place to see loop: Execute these on the counter, upto some number
  • Meaning: Execute these on the counter while counter <= number
  • counter must be initialized, usually to 0 or 1
  • The counter must be incremented within the loop
    • counter = 0
while counter <= 10:
    print("Counting from 0 up to 10", counter)
    counter = counter + 1   # incrementing counter
  • Example 1: print 'A', print 'B', print 'C', until answer == 'Yes!'
  • Translate to python: 
    
answer = input('Do you like Python? (You MUST say Yes)' )
while not (answer == 'Yes'):
    print('Python is wonderful!')
    print('Python is awesome!')
    print('You should like Python!')
print('Done')
  • !!! ERROR !!! Your script never finishes
  • answer is asked once, but is not updated
  • The code block continues to run, infinitely: infinite loop
  • Fix:

answer = input('Do you like Python? (You MUST say Yes)' )
while not (answer == 'Yes'):
    print('Python is wonderful!')
    print('Python is awesome!')
    print('You should like Python!')
    answer = input('Do you like Python? ')  # This line is here to update answer
print('Done')

Loop example

Write a script to ask a user to enter a number larger than 10, then start counting upto that number

  • Get user's input as end_number, and convert that to an int
  • Initialize a count to 1
  • Start counting, until count is upto user's input
  • Update count so we do not have an infinite loop

answer = input('Please enter a number larger than 10:')
end_number = int(answer)
count = 1
while count <= end_number:
    print(count)
    count = count + 1
print('done')

Loop (cont.)

  • A loop continues to execute, unil the while condition is False
  • We can 'prematurely' exit the loop using break
  • We can 'prematurely' restart the loop using continue
Base script

num = 0
while num < 10:
    print('The number is', num)
    num = num + 1 # increment num, using the current value
    print('The next number is', num)
print('done')
What if we add a break

num = 0
while num < 10:
    print('The number is', num)
    num = num + 1 # increment num, using the current value
    break
    print('The next number is', num)
print('done')
What if we add continue

num = 0
while num < 10:
    print('The number is', num)
    num = num + 1 # increment num, using the current value
    continue
    print('The next number is', num)
print('done')

Nested loop

  • Just like nested branch, you can have nested loop
  • Nested loops are not common, and generally discouraged due to performance implications
  • You can nest multiple levels of loops

round = 0
while round < 5:
    print('-----------------------------------')
    print('Counting from 1 to 3, round', round)
    round = round + 1
    count = 0
    while count < 3:
        print(count)
        count = count + 1

Nested branch in a loop

  • You can nest a branch within a loop
  • This is usually used to break or continue when certain condition is match
  • This is also used to perform 'special cases' within a loop

Count from 0 to 100, but stop after a number specified by the user

  • Get user's answer, convert to an int, and store in a variable named end
  • Initialize the counter to 0
  • Start counting, until counter is up to 100
  • Within the loop: Update the counter by incrementing
  • Within the loop: Check if counter > end then break

answer = input('Enter the stop number: ')
end = int(answer)
counter = 0
while counter <= 100:
    print('Counting number:', counter)
    counter = counter + 1
    if counter > end:
        print('I am stopping after', end)
        break
print('done')

Count from 0 to 100, and print if the current number is odd or even

  • Don't need any input from user
  • Initialize a counter to 0
  • Start counting, upto 100
  • Within the loop: If the counter is even, print even, else, print odd
  • Within the loop: Update the counter by incrementing

counter = 0
while counter < 100:
    if counter % 2 == 0:
        print(counter, "is an even number")
    else:
        print(counter, "is an odd number")
    counter = counter + 1

Nested loop in a branch

  • Usually used to validate certain conditions before executing the loop
  • Example: Count from a number specified by the user to 50
  • Get a start_number as an int from user's input
  • If start_number < 50:
    • initialize counter to start_number

answer = input('Tell me where to start')
