Object Oriented Programming (OOP)

OOP theory

  • OOP: Programming methodology based on classes and objects
  • Most programming problems are solved using functions/procedures/methods and data
  • The methods and data are closely related to a well defined problem
  • For example:
    • Calculate a person tax
    • Data: income, deductibles, investments, etc
    • Methods: tax rate based on income, percentage of return on investment, etc
    • Calling tax methods with tax data to calculate individual tax

  • OOP suggests we redesign our program around individual tax
  • Create a class called IndividualTax
  • The class contains generic variables, and functions
  • We then create a specific instance of a class, called an object
  • For example: my_tax is an instance of IndividualTax class
  • my_tax contains specific values for each variable
  • my_tax uses the same methods available in the IndividualTax

  • All classes have a constructor and destructor
  • Constructor initializes data as the object is instantiated, or created
  • Destructor clean things up as the object is deleted

  • Classes provides encapsulation
  • Class data (variables) can be hidden and isolated from the public (similar to variable scope)
  • A class can provide getter and setter to get and set its data.
  • This provide validation and protection for the class data

  • Classes have inheritance: A class can be a child of another class
  • The sub-class inherits super-class methods and data
  • The super-class does NOT have access to the sub-class methods and data

For example:

  • A Vehicle class has a variable color
  • A Car class is a sub-class of Vehicle.
    • It has access to color variable
    • It has a variable nbr_of_doors
    • Vehicle does NOT have access to nbr_of_doors
  • A Bicycle is also a sub-class of Vehicle
    • It also have access to color variable
    • It does NOT have access to the Car's nbr_of_doors variable

  • Sub-class may modify the super-class behavior (by modifying the super-class methods)
  • This is called polymorph
  • For example:
  • A Vehicle class has a function called get_wheels_nbr(), which is abstracted, and not implemented
  • A Truck implements the get_wheels_nbr() to return 4
  • A Motorcycle implements the get_wheels_nbr() to return 2

OOP in Python

Syntax: 

class MyClass(object):                       # Define MyClass
    def __init__(self, data1):                     # constructor
        self.variable1 = data1                     # initialize variable
    def __del__(self):                             # destructor
        cleanup_this()
    def do_something(self, data3):                 # a class method
        if self.data1 > data3:
            print('Greater')
        else:
            print('Not greater')
        return

obj1 = MyClass(1)                                  # initialize an object
obj1.do_something(3)                               # print Not greater
obj1.variable1 = 4                                 # Modify variable1
obj1.do_something(3)                               # print Greater

  • Python does not really have encapsulation
  • The base class for all python classes is object
  • By convention: class name is Capitalized
  • To create an object of a class, invoke the class name like a function
  • The first argument to all methods is self, which references the current object

Inheritance example 
class MyBase(object):
    def __init__(self, number):
        self.base_number = number
    def print_number(self):
        print('Number is', self.base_number)
class MyChild1(MyBase):
    def __init__(self, number):
        self.base_number = number + 10
class MyChild2(MyBase):
    def print_number(self):
        print('My number is changed to:', self.base_number + 100)
obj1 = MyBase(1)
obj1.print_number()     # print 1
obj2 = MyChild1(1)
obj2.print_number()     # print 11
obj3 = MyChild2(1)
obj3.print_number()     # print 101

  • Without specifying a constructor, the current class will use its super-class constructor
  • By defining a funcition with the same name, you can archive polymorphism
  • But...how do I access the parent's function?
  • Use the super() keyword:
    • First argument to super() is the current class name
    • Second argument to super() is self
    • Super returns the parent object
    • You can access super-class data and methods

super example: 

class MyBase(object):
    def __init__(self, number):
        self.number = number
    def print_it(self):
        print('number is', self.number)
class MyChild(MyBase):
    def print_it(self):
        super(MyChild, self).print_it()         # calls the parent's method
        print('Then, my number is changed to:', self.number + 10)
obj = MyChild(1)
obj.print_it():                 # print:      number is 1
                                # also print: Then, my number is changed to: 11

Procedural vs. OOP

def get_name():
    return input('Please enter your name')
def greet(name):
    print('Special greeting to', name)

name = get_name()
while not name == 'done':
    greet(name)
    name = get_name()

OOP design: 

class MyGreeter(object):
    def __init__(self):
        self.name = 'done'
    def get_name(self):
        self.name = input('Please enter your name')
    def greet(self):
        print('Special greeting to', name)
    def run(self):
        self.get_name()
        while not self.name == 'done':
            self.greet()
            self.get_name()

greeter = MyGreeter()
greeter.run()
